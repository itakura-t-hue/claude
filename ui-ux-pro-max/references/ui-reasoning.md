# UI Design Decision Matrix

A comprehensive design reference for 40+ product types with recommended patterns, color psychology, typography guidance, and anti-patterns to avoid.

---

## 1. SaaS (B2B Dashboard)

| Aspect | Recommendation |
|--------|---|
| **Style Pattern** | Minimalist, data-focused, professional grid-based layout |
| **Color Mood** | Cool blues, neutral grays, accent with teal/green for actions |
| **Typography Mood** | Geometric sans-serif (Inter, Roboto), high contrast, readable at small sizes |
| **Key Effects** | Subtle shadows for depth, hover states on data, smooth transitions (200-300ms) |
| **Anti-patterns** | Excessive gradients, bright colors, ornamental elements, auto-animating charts |
| **Severity** | HIGH - Clarity and quick scanning are critical |

## Quick Reference: Severity Guidelines

- **HIGH Severity**: Poor design directly impacts core metrics (conversion, retention, legal compliance, safety)
- **MEDIUM Severity**: Design affects experience and engagement but has workarounds
- **LOW Severity**: Design is nice-to-have but not critical

---

## Decision Framework

When choosing a design pattern:

1. **Identify product type** - Match to closest category above
2. **Assess primary metric** - Conversion? Retention? Clarity? Efficiency?
3. **Check user context** - Desktop/mobile/both? Speed-critical? Accessibility essential?
4. **Apply recommendations** - Use color mood and typography mood as guidance
5. **Audit for anti-patterns** - Specifically test against listed anti-patterns
6. **Test with severity in mind** - HIGH severity items require multiple validation rounds

---

## Additional Notes

- Color recommendations are starting points; always test with your brand guidelines
- Typography mood can be achieved through font choice OR weight/size emphasis
- Key effects should enhance (not replace) clear information architecture
- Mobile variants may require pattern simplification
- Dark mode adaptations are increasingly table-stakes for modern products
