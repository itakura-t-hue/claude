# AIっぽさ排除ガイド

AIが生成するUI/UXには特徴的なパターンがあり、デザイナーや開発者が見ると「これAIだな」とすぐわかる。このガイドはその典型的なパターンを潰し、人間のデザイナーが作ったかのような自然なUIを生成するためのルール集。

**これは全てのUI生成タスクで最初に参照すべきファイル。**

---

## 1. テキスト・コピーライティング

### AIが書きがちなNGフレーズ

以下のフレーズはAIの「手グセ」。使ったら即バレる。

| NGフレーズ | なぜダメか | 代替案の例 |
|-----------|----------|-----------|
| 「一気通貫で」 | AIが好む四字熟語的表現 | 「まるごと」「全部まとめて」「最初から最後まで」 |
| 「体系的に習得」 | 教科書的で血が通ってない | 「順番に身につく」「ステップバイステップで覚える」 |
| 「包括的な」 | 抽象的すぎて何も伝わらない | 具体的な数字や範囲を書く（「12カテゴリの」「フロントからバックまで」） |
| 「最適な」「効果的な」 | 曖昧な修飾語の連発 | 具体的にどう良いか書く |
| 「シームレスな体験」 | テックバズワードの空打ち | 何がスムーズなのか具体的に |
| 「革新的な」「先進的な」 | 自画自賛のAI臭 | 実際の機能や違いを語る |
| 「〜を実現します」 | セールスレターのテンプレ | 「〜できる」「〜になる」とシンプルに |
| 「プロフェッショナルな」 | 中身がないのに格好つけてる | 削除するか、何がプロなのか具体化 |
| 「あらゆる〜に対応」 | 大風呂敷。嘘に聞こえる | 対応範囲を具体的に列挙 |
| 「データドリブンな」 | バズワード | 「GA4の数字を見ながら」等、具体ツール名で |

### 良いコピーの書き方

- **口語寄り**にする。LP向けなら話しかけるように。「まずはここから始めよう」のような表現。
- **具体的な数字**を入れる。「多くのスキル」→「12カテゴリ、全29章」
- **ターゲットの言葉**で書く。開発者向けなら技術用語OK、一般向けなら噛み砕く
- **「何ができるようになるか」**をストレートに。回りくどい修飾語は削る
- **短い文**を心がける。一文に情報を詰め込みすぎない
- セクション説明が全て「〜を習得します。」で終わるのは単調。語尾を変える

## 2. レイアウト・構造

### AIが作りがちなNG構造

**最大のNG: 「見出し → 説明文 → 3カラム均等グリッド」の永遠ループ**

```
[セクションA]
  h2 + p + 3col grid (card, card, card)

[セクションB]
  h2 + p + 3col grid (card, card, card)

[セクションC]
  h2 + p + 3col grid (card, card, card)
```

これが3回以上続くと、人間は「テンプレートだな」と気づく。

### 解決策: セクション構成にバリエーションを持たせる

以下のパターンを組み合わせて、セクションごとに構造を変える:

**パターン1: 全幅ビジュアル + テキストオーバーレイ**
```
[full-width image/video]
  [overlay: h2 + short text]
```

**パターン2: 左右分割（テキスト + ビジュアル）**
```
[left: h2 + p + CTA]  [right: screenshot/photo/illustration]
```
→ 次のセクションでは左右を入れ替える

**パターン3: ベントグリッド（異なるサイズのカード）**
```
[large card spanning 2col]  [small card]
[small card]                [medium card]
```

**パターン4: タイムライン/ステッパー**
```
1 ──── 2 ──── 3 ──── 4
```

**パターン5: 全幅テキスト（大きなステートメント）**
```
[center: 大きいフォントの1行メッセージ]
```

**パターン6: スクロール連動ビジュアル**
```
[sticky sidebar image] + [scrolling text blocks]
```

**ルール**: 連続する2つのセクションが同じレイアウト構造にならないようにする。

### グリッドの崩し方

- **カードサイズを混ぜる**: 全て同じサイズにしない。1つ大きいカード + 2つ小さいカード、など
- **余白に変化をつける**: セクション間のpaddingを全て同じにしない（80px → 120px → 60px のように揺らす）
- **非対称にする**: 左右完全対称は機械的。片側を少し大きく、テキスト配置をずらす
- **はみ出し要素**: 画像やカードが親コンテナから少しはみ出す演出（negative margin、translateX）
- **角度**: 要素を微妙に傾ける（2-3deg）、セクション区切りに斜めのdivider

## 3. ビジュアル素材

### AI生成UIの最大の弱点: 写真・イラストがない

AIは著作権を気にしてビジュアル素材を入れたがらない。結果、テキストとSVGアイコンだけの「のっぺりした」画面になる。

**これが最もAIっぽく見える原因。**

### 解決策

**写真を使う場合:**
- Unsplashの画像URLを直接埋め込む: `https://images.unsplash.com/photo-XXXXX?w=800&q=80`
- Pexelsも可: `https://images.pexels.com/photos/XXXXX/pexels-photo-XXXXX.jpeg?w=800`
- `<img>` にはalt属性を必ず入れる
- アスペクト比を指定して CLS を防ぐ

**プレースホルダーを使う場合:**
- 明確に「ここに実際の写真/スクリーンショットを入れる」とコメントで示す
- CSSだけのプレースホルダーを作る場合も、gradient + textで「IMAGE PLACEHOLDER」と表示する
- 絶対に空白のまま放置しない

**どこにビジュアルを入れるか:**
- ヒーローセクション: 背景画像 or プロダクトのスクリーンショット
- Features: 各機能のスクリーンショット or アイコン+写真のハイブリッド
- Testimonial: ユーザーのアバター写真
- CTA: 背景にムードフォト
- About/Team: 人物写真
- Case Study: ビフォー/アフターのスクリーンショット

**最低でもページ内に2-3箇所はビジュアル素材を入れること。**

## 4. タイポグラフィの「くずし」

### AIが作りがちなNGタイポグラフィ

- 見出しも本文も同じフォントファミリー
- フォントサイズの段階が教科書的（h1:48px → h2:36px → h3:24px → p:16px と均一）
- 全てのテキストが同じウェイト（400 or 700だけ）
- レタースペーシングやラインハイトがデフォルトのまま

### 解決策

- **見出しに個性を出す**: 超大きいフォントサイズ（80px+）を1箇所使う、全角英数を混ぜる
- **フォントウェイトの幅を広げる**: 300（細い説明文）、400（本文）、600（小見出し）、800-900（大見出し）
- **アクセントとしてのモノスペース**: 数字、コード、ラベルにモノスペースフォントを混ぜると「デザインされた感」が出る
- **サイズのジャンプを大胆に**: h1は思い切って大きく、pとの差を開ける
- **色で抑揚をつける**: 見出しの一部分だけアクセントカラーにする（「身につく**スキル**」の「スキル」だけ色変え）

## 5. カラーの「くずし」

### AIが作りがちなNGカラー

- primary / secondary / accent を教科書通りに3色だけ使う
- ボタンはprimary、見出しはsecondary、アクセントは...みたいにルール通りすぎる
- 背景が真っ白（#fff）or 真っ黒（#000）のベタ塗り

### 解決策

- **背景にわずかな色味**: 真っ白ではなく#FAFBFC、真っ黒ではなく#0a0a0f のように微妙な色味を入れる
- **セクションごとに背景色をわずかに変える**: 全セクション同じ背景色は単調。微妙に色温度を変える
- **グラデーションの自然な使い方**: ボタンだけでなく、セクション背景にも超淡いグラデーション
- **テキストカラーを少しバラす**: 説明文に#333と#555を混ぜる（重要度で）
- **写真の色味をデザインに取り込む**: ヒーロー写真の支配的な色をアクセントに使う

## 6. インタラクションの「人間味」

### AIが作りがちなNG

- 全てのhoverエフェクトが `translateY(-4px)` + `box-shadow` の同一パターン
- 全カードのアニメーションが同時発火（delay差がない、あっても0.1s刻みで機械的）
- スクロールアニメーションが全部 fadeInUp

### 解決策

- **hover のバリエーション**: カードは `translateY` だけど、ボタンは `scale`、リンクは `color` + `underline` のように要素種別で変える
- **アニメーション delay に「遊び」**: 0.1, 0.15, 0.25, 0.4s のように不均一にする。等差数列にしない
- **スクロール方向を混ぜる**: fadeInUp だけでなく、fadeInLeft（左から）、scaleIn（拡大）、clipReveal（マスク解除）を使い分ける
- **マイクロインタラクション**: ボタンクリック時のripple、input focus時のlabel浮き上がり、scroll progressバー、ページ遷移トランジション
- **動きの「ため」**: ease-out だけでなく cubic-bezier(0.34, 1.56, 0.64, 1)（オーバーシュート）など個性的なイージングを1-2箇所入れる

## 7. 「テンプレ感」を消すためのディテール

### 細部で差がつくポイント

- **装飾線を1-2箇所**: セクション区切りに細い線 + 小さなアイコン/ドット
- **引用符やマーカー**: テキストの横に縦線（border-left）、数字の横にドット
- **背景パターン**: ドット、グリッド線、ノイズテクスチャをごく薄く（opacity 0.02-0.05）
- **手描き風の要素**: 下線に波線、矢印にSVG手描き風パス（1箇所でも印象が変わる）
- **実際のデータっぽい表示**: 「1,234ユーザー」「98.5%の満足度」のような具体的な数字
- **テスティモニアル**: 架空でもいいのでユーザーの声を入れる。名前、肩書き、写真（placeholder可）

## 8. チェックリスト

生成したUIを納品する前に、以下を全てチェック:

- [ ] 3カラム均等グリッドが2回以上連続していないか
- [ ] 連続するセクションが同じレイアウト構造になっていないか
- [ ] テキストにAI定型フレーズ（上記NGリスト）が残っていないか
- [ ] ページ内に最低2-3箇所のビジュアル素材（写真/イラスト/スクショ）があるか
- [ ] カードサイズ・余白が全て同じになっていないか
- [ ] セクション間のpaddingに変化があるか
- [ ] hover効果が全て同じパターンになっていないか
- [ ] スクロールアニメーションの方向にバリエーションがあるか
- [ ] 見出しの一部にアクセントカラーや異なるウェイトが使われているか
- [ ] 背景色がベタ塗りではなく微妙な色味/グラデーションがあるか
- [ ] コピーが口語的で具体的か（「〜を実現します」のような営業トークになっていないか）
- [ ] どこか1箇所に「予想外」の要素（大胆なフォントサイズ、はみ出し画像、斜めdivider等）があるか
