# 競合調査プロンプト

## 入力

- サイトURL: {URL}

## 実行手順

以下のPhase順に実行すること。各Phaseの完了後、次に進む前に結果を簡潔に報告すること。

---

### Phase 1: 対象サイトの基本理解（Ahrefs不要）

Web検索・サイト閲覧で以下を把握する:
- サービス内容・提供価値・ターゲット顧客
- 主要な機能・料金プラン
- サイト構造（主要カテゴリ・コンテンツの種類）
- 事業ドメイン・解決している課題

→ この情報をもとに「対象サイトが狙っていると推定されるキーワード」を10個リストアップする。

---

### Phase 2: SEO現状把握（Ahrefsクエリ 1〜2回目）

**クエリ1: Site Explorer - ドメイン概要**
- 対象: {URL}
- 取得項目: DR（ドメインレーティング）、オーガニックトラフィック、被リンク数
- ツール: `site-explorer-domain-rating` + `site-explorer-organic-keywords`（上位キーワード10件、トラフィック順）

**クエリ2: Site Explorer - オーガニック競合**
- 対象: {URL}
- 取得項目: オーガニック検索の競合ドメイン上位10件
- ツール: `site-explorer-organic-competitors`（上位10件、共通キーワード数順）

→ Phase 1の推定キーワードとPhase 2の実データを突合し、主要ターゲットキーワードを5〜8個に絞り込む。

---

### Phase 3: キーワード深掘り（Ahrefsクエリ 3回目）

**クエリ3: Keywords Explorer - キーワード概要**
- 対象: Phase 2で絞り込んだ5〜8個のキーワード（カンマ区切りで一括取得）
- 取得項目: 検索ボリューム、KD（キーワード難易度）、CPC、トラフィックポテンシャル
- ツール: `keywords-explorer-overview`

---

### Phase 4: 競合の定量比較（Ahrefsクエリ 4〜5回目）

Phase 2のオーガニック競合から、SEO競合として上位5サイトを選定する。

**クエリ4-5: Site Explorer - 競合ドメインの概要**（最大2回に分割）
- 対象: 選定した競合5サイト
- 取得項目: 各サイトのDR、オーガニックトラフィック
- ツール: `site-explorer-domain-rating` を効率的に使用

---

### Phase 5: サービス競合調査（Ahrefs不要）

Web検索で以下を調査する:
- 対象サイトと同じ課題を解決するサービスを5つ特定（SEO競合と重複OK）
- 各サービスについて:
  - サービス概要・主要機能
  - 料金プラン（公開情報がなければ「非公開」と記載）
  - ターゲット顧客像
  - 差別化ポイント

---

### Phase 6: 総合分析・レポート作成（Ahrefs不要）

Phase 1〜5の情報を統合し、以下のMarkdownレポートを出力する。

---

## ツール利用ルール

### Ahrefsクエリ計画（合計5回以内）

| # | ツール | 対象 | 取得内容 |
|---|--------|------|---------|
| 1 | site-explorer-domain-rating + organic-keywords | 対象URL | DR・トラフィック・上位KW |
| 2 | site-explorer-organic-competitors | 対象URL | 競合ドメイン一覧 |
| 3 | keywords-explorer-overview | 絞込済KW群 | ボリューム・KD・CPC |
| 4 | site-explorer-domain-rating | 競合1〜3 | DR・トラフィック |
| 5 | site-explorer-domain-rating | 競合4〜5 | DR・トラフィック |

- **実行前に必ず**: 「次にAhrefsで○○を取得します（クエリ○回目/5回）」と宣言し、ユーザーの許可を得ること
- **5回を超える場合**: 追加の必要性を説明し、許可を得ること
- **Web検索**: サイト情報・サービス調査に自由に使用可

### データ品質ルール
- 推測で補った情報には「※推定」を明記
- 料金等の公開情報が不明な場合は「非公開」と記載
- 分析の根拠（参照URL・データソース）を各セクションに付記
- Ahrefsデータの取得日を明記
