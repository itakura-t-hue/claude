# 第1部: 競合調査レポート

## 1. サイト概要

### サービス内容

🔍 Fortinet（フォーティネット）は、統合サイバーセキュリティプラットフォーム「Fortinet Security Fabric」を中核に、ネットワーク・エンドポイント・クラウドを横断するセキュリティソリューションを提供するグローバル企業。主力製品はFortiGate（次世代ファイアウォール / UTM）で、独自開発のASICプロセッサ搭載により高いコストパフォーマンスを実現している。

### 主要製品群

| 製品 | カテゴリ |
|------|---------|
| FortiGate | 次世代ファイアウォール / UTM（主力製品）|
| FortiSASE | クラウド型セキュアアクセスサービスエッジ |
| FortiEDR / FortiClient | エンドポイントセキュリティ |
| FortiManager / FortiAnalyzer | 集中管理・分析 |
| FortiSwitch / FortiAP | ネットワークインフラ |
| FortiGuard Labs | 脅威インテリジェンスサービス（サブスクリプション型）|

### ターゲット顧客

🔍 確認済みの事実:
- **大企業・政府機関・金融機関** — エンタープライズ向けハイエンドアプライアンス
- **中堅・中小企業** — FortiGate エントリーモデルによるUTM
- **OT/産業セキュリティ分野** — 製造業・重要インフラ（2025年にOT Security Platform拡張を発表）

### ビジネスモデル

🔍 ハードウェアアプライアンス販売 + サブスクリプション型セキュリティサービス（FortiGuard）の二本柱。UTP / Enterprise / 360 Protectionの3つのライセンスバンドルで段階的に機能を拡充。日本市場ではSB C&S、日立ソリューションズ等のパートナー経由で販売。

> 💡 このデータは電通月次レポート（Slide 28「LLMOの評価」）の「CISCOのメンション数が継続して伸長、Fortinet・Palo Altoは横ばい」という分析とも整合する。ただし電通レポートは「メンション」のみを追跡しており、「引用」の分析は含まれていない。引用データはメンションよりもコンテンツ施策の効果を直接反映する指標であり、Brand Radarの引用タブを月次で追跡することで、施策の効果をより正確に測定できる。

#### 15-Y-5. Brand Radarの観測方法論上の限界と独自ツールの位置づけ

**限界1: プロンプトの固定性**

Brand RadarはGoogle検索のPAA（People Also Ask）から質問を収集し、それをLLMに投げてメンション・引用をカウントしている。つまり観測対象の「質問」がPAA起点で固定されており、ユーザーが実際にLLMに投げる質問とは異なる可能性がある。

- 例: 「FortiGateとPalo Altoどっちがいい？」のような比較検討系の質問はPAAに出にくいが、ChatGPTには頻繁に投げられる
- 例: 「UTMって本当に必要？中小企業向けのおすすめは？」のような導入検討系もPAAとは文脈が異なる
- カスタムプロンプト機能はAPI上存在するが（`prompts: "custom"`）、現在のレポートでは未設定（空配列）

→ **対応:** カスタムプロンプトを設計し、比較検討系・導入検討系・トラブルシューティング系の質問セットを追加する。これによりPAA起点では捕捉できない「購買に近い質問」でのブランド露出を観測できるようになる。

**限界2: ボリューム推定の間接性**

Brand Radarの「Volume」はPAAの出現頻度×関連KWの検索ボリュームからの推定値であり、**実際にLLMにその質問が投げられた回数ではない**。Google検索データからの間接推定なので、LLMへの実問い合わせ数とは乖離がある。

→ 現時点で対応策はない（LLMプロバイダー側がデータを公開していないため）。この制約は把握した上で数値を解釈する必要がある。

**限界3: センチメント分析の欠如**

Brand Radarはメンション数・引用数を定量的にカウントするが、**その文脈がポジティブかネガティブかの判定はしない**。

- 「Fortinetは高コスパで業界No.1」→ ポジティブメンション
- 「Fortinetは脆弱性が多く注意が必要」→ ネガティブメンション
- 「Fortinetの競合であるPalo Altoがおすすめ」→ Fortinetメンションだが実質ネガティブ

これらが全て同じ「1メンション」としてカウントされる。

→ **対応:** Brand Radarの`ai-responses`エンドポイントから回答テキスト（`response`フィールド）を取得し、独自のセンチメント分析を実施。ポジティブ/ネガティブ/中立の分類を行い、「メンション865件のうちポジティブは何件か」を追跡する。

**限界4: 「なぜ引用されないか」の因果分析ができない**

Brand RadarはLANSCOPEが3,800引用でFortinetの2.7倍と教えてくれるが、**なぜLANSCOPEのコンテンツが選ばれてFortinetが選ばれないのか**の分析はできない。因果関係の特定には:
- コンテンツの構造分析（構造化データの有無、定義文の明確さ）
- 更新頻度の比較
- ドメインのローカルシグナル（.jp vs /jp）
- コンテンツの独自性（一次情報の有無）

等を突合する必要があるが、これはBrand Radarの守備範囲外。

→ **対応:** ソースコード分析（本レポート15-Xで実施済み）やコンテンツ比較分析を独自に行い、Brand Radarの数値データと突合する。

**限界5: LLM応答の非決定性**

LLMの回答は同じプロンプトでも毎回変わる。Brand Radarがいつ・何回そのプロンプトを投げたかの情報はなく、1回のスナップショットなのか複数回の平均なのかが不明。「Fortinetのメンション865件」は特定タイミングでの観測結果であり、再現性の保証がない。

→ **対応:** 独自ツールで同一プロンプトを週次で複数回投げ、回答のばらつき（分散）を測定する。これにより「安定してFortinetが言及される質問」と「不安定な質問」を切り分けられる。

**補足: Geminiの引用データがゼロについて**

Brand Radarの引用タブでGeminiが全ブランド0件だが、これはBrand Radarの問題ではなく**Geminiの仕様**。Geminiは回答時にソースURLを引用リンクとしてユーザーに提示しない（Google検索のGroundingで内部的に参照はしているが、引用として表出しない）。したがってGeminiの引用がゼロなのは正常動作であり、データの信頼性の問題ではない。Geminiについてはメンションタブ（Cisco 64、Fortinet 23等）のみで評価する。

**限界7: search_queriesの取得制限**

API仕様上、`search_queries`フィールド（AIがどんな検索クエリから回答を生成したか）はChatGPTとPerplexityのみ対応。AI Overviews / AI Mode / Gemini / Copilotでは空になる。つまりAIOで「どんなKWでの検索がFortinetの引用に繋がったか」のKW単位分析はBrand Radarではできない。

→ **対応:** AhrefsのSERP分析（serp-overview）でKW単位のAIO引用先を取得し、Brand Radarのプラットフォーム別データと突合する。

#### 15-Y-6. 独自観測ツールの位置づけ — Brand Radarを補完する3つの機能

Brand Radarの上記限界を踏まえ、以下の3機能を独自ツールとして提供することで、AI露出分析の精度を向上させる。

**機能1: カスタムプロンプトによる定点観測**

Brand Radarの固定プロンプト（PAA起点）では捕捉できない「購買に近い質問」を独自に設計し、主要LLM（ChatGPT、Gemini、Perplexity）に定期的に投げて回答を記録する。

対象プロンプト例:
- 「中小企業向けのUTM製品でおすすめはどれですか？」
- 「FortiGateとPalo Altoのファイアウォールを比較してください」
- 「ゼロトラストを導入したいが、どのベンダーが良いか」
- 「FortiClientは今後もサポートされますか？」
- 「サイバーセキュリティ対策で最初に導入すべき製品は何ですか」

→ 月次でこれらのプロンプトへの回答を記録し、Fortinetが推奨されているか・引用されているか・競合との比較でどう評価されているかを追跡する。

**機能2: 回答テキストのセンチメント分析**

Brand Radarから取得した回答テキスト（`ai-responses`の`response`フィールド）+ 独自プロンプトの回答テキストに対して、センチメント分析を実施。

分類基準:
- **ポジティブ:** Fortinetを推奨・評価する文脈（「コスパが高い」「市場シェアNo.1」等）
- **ネガティブ:** Fortinetを否定・注意喚起する文脈（「脆弱性が多い」「サポートに不安」等）
- **中立:** 比較表の一項目として言及される等
- **競合推奨:** Fortinetに言及しつつも他社を推奨する文脈

→ メンション865件の内訳を「ポジティブ○%/ネガティブ○%/中立○%」として定量化。月次でセンチメント構成比の変動を追跡する。

**機能3: 引用要因の因果分析レポート**

Brand Radarの「LANSCOPEが3,800引用でFortinetの2.7倍」というデータに対して、**なぜその差が生まれているのか**を分析するレポートを月次で作成。

分析項目:
- LANSCOPEのAIO引用ページ vs FortinetのAIO引用ページの構造比較（構造化データ、H構造、文字数、更新日）
- 引用されているページと引用されていないページの共通点・差異
- コンテンツ改善後の引用数変動（ピラーページ化の効果検証）

→ Brand Radarの「何が起きているか」に、独自分析の「なぜ起きているか」を加えることで、施策の優先度判断を支援する。

> 💡 **まとめ:** Brand Radarはメンション・引用・SoVの定量追跡に優れているが、「プロンプトの固定性」「センチメント分析の欠如」「因果分析の不在」という3つの限界がある。独自ツールはこの3つの穴を埋める補完的な位置づけであり、Brand Radarを置き換えるものではない。Brand Radar（定量追跡）+ 独自ツール（定性分析）の2層体制で、AI露出の包括的な観測を実現する。

## 17. まとめ — 先方への説明のポイント

### 伝えるべきメッセージ

1. **現行施策は成果が出ている。** 流入は前年比+11.5%で成長中。一般KWの記事施策は継続すべき。

2. **ただし、流入の中身が変わった。** 「調べもの層」（用語集）が増え「製品検討層」（指名検索）が減った。これがCV低下の主因。

3. **原因はAI Overviewによる検索行動の変化。** 指名検索でもAIOが表示され、クリックなしで情報が得られる環境になった。これは業界全体のトレンドであり、Fortinet固有の問題ではない。

4. **対策は3本柱。**
   - ①コンテンツ施策の継続（AIO対応の構造化を追加）
   - ②指名検索のCTR回復（meta description最適化、forticlient PAA対応）
   - ③サイト内導線の最適化（直帰率改善、CTA強化）

   いずれもSEOの範囲内で実行可能。

5. **コンテンツ投資の二重効果を活用する。** 一般KWで上位化したコンテンツは、AIOの情報源になることで中長期的に指名検索の獲得に繋がる。短期のトラフィック +中期の指名検索獲得の両方にリターンがある。

6. **AIOの状況を定期的に観測する体制を整える。** これはLLMO施策ではなく、SEOの効果を正しく測るための指標追加。追加コストはほぼゼロ。
