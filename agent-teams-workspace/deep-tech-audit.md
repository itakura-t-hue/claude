### ローカルSEOの詳細所見

**良い点:**
- 全院ページでtitle・H1に地域名が含まれている
- meta descriptionにも院名（地域名含む）が記載
- MedicalClinicスキーマが全院に実装されている
- GeoCoordinates（緯度・経度）が設定されている
- NAP情報がページ上に表示されている

**問題点:**

1. **telephone属性が共通フリーダイヤル（+81120127411）** — 各院の個別電話番号ではなくフリーダイヤルが構造化データに設定されている。Googleマイビジネスとの整合性に問題が生じる可能性。

2. **openingHoursSpecificationが空値** — opens/closesの時間が空白のまま設定されており、構造化データとして無効。実際の営業時間（例: 10:00-20:00）がスキーマに反映されていない。

3. **Productスキーマのprices有効期限が過去日** — priceValidUntilが `2025-12-31` となっており、すでに期限切れ。

## 3. PageSpeed Insights

**結果:** Google PageSpeed Insights APIがレート制限（HTTP 429）により取得不可。

**確認済み事実（代替確認）:**
- viewportメタタグの明示的な設定は未確認だが、CSSメディアクエリ（`max-width: 767px`）によるレスポンシブ対応は実装済み
- 画像のlazy loading (`loading="lazy"`) は未使用
- 画像のalt属性は一部のみ設定（ロゴ等は設定あり、装飾画像は未設定）

### 4-2. 内部リンク構造の所見

**確認済み事実:**

1. **FAQの二重構造** — `/faq/phimosis/` と `/phimosis/faq/` のように、同じカテゴリのFAQが2つの異なるURL構造に存在。カニバリゼーションの可能性が高い。

2. **sitemapの更新日が古い** — サイトマップ内の多くのページのlastmodが2020-2021年で止まっている。実際にはページが更新されているにもかかわらず（WebPage schema上のmodifiedDateは2026年）、サイトマップが更新されていない。

3. **リンク階層は概ね3階層以内** — トップ → カテゴリ → 個別ページの構造で、過度に深い階層は確認されなかった。

4. **トップページからの直接リンク数は100+** — 全国の院ページへのリンクがフッター/ナビゲーションに含まれており、トップページのリンク数が非常に多い。

5. **外部サイトへのリンク** — `houkei.tokyo.jp` への外部リンクがトップページに存在（関連サイトと推測されるが確認不可）。

## 5. robots.txt / サイトマップ分析

### robots.txt
```
User-Agent:*

Sitemap: https://www.norst.co.jp/sitemap.xml
```

**問題点:**
- `User-Agent:*` の後にコロンとアスタリスクの間にスペースがない（`User-Agent: *` が正式な書式）
- Disallow指示がないため全ページがクロール許可されているが、`/reserve/` や `/contact/` のようなフォームページのクロール制御がない

### サイトマップ
- 389 URL を含む
- lastmodが2020-2021年で停止しており、実態と乖離
- 個別FAQページ（faq001.html〜faq167.html）が167ページ含まれている

## 7. 問題点一覧と改善アクション

### 深刻度S（最優先）

#### S-1. FAQページにFAQPageスキーマが未実装
- **確認済み事実:** `/faq/` ページにFAQPage構造化データが存在しない
- **影響:** FAQ系キーワードでリッチリザルト（アコーディオン表示）が表示されない
- **改善アクション:**
```json
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "包茎手術は痛いですか？",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "テープ式麻酔やジェル麻酔を使用し、痛みを最小限に抑えます。"
      }
    }
  ]
}
```

#### S-2. 全ページでOGPタグが未実装
- **確認済み事実:** 分析した全11ページでog:title, og:description, og:image等が未設定
- **影響:** SNS共有時にサムネイルやタイトルが正しく表示されない。LINEでの共有が多い業種で特に影響大
- **改善アクション（トップページ例）:**
```html
<meta property="og:title" content="包茎手術の東京ノーストクリニック【公式】">
<meta property="og:description" content="包茎手術専門。開院35年の実績。全国36院展開。24時間予約受付。">
<meta property="og:type" content="website">
<meta property="og:url" content="https://www.norst.co.jp/">
<meta property="og:image" content="https://www.norst.co.jp/wp/wp-content/themes/norst/images/ogp.jpg">
<meta property="og:site_name" content="東京ノーストクリニック">
<meta name="twitter:card" content="summary_large_image">
```

#### S-3. トップページにH1タグが存在しない
- **確認済み事実:** トップページでH1タグが検出されなかった
- **影響:** ページの主題をGoogleに伝える最重要の見出しタグが欠如
- **改善アクション:** メインビジュアル内またはページ上部に適切なH1を設定
```html
<h1>包茎手術の東京ノーストクリニック</h1>
```

### 深刻度A（重要）

#### A-1. meta descriptionの重複・未設定
- **確認済み事実:**
  - `/faq/` のmeta descriptionがトップページと完全に同一（FAQ固有の内容になっていない）
  - `/reserve/` にmeta descriptionが未設定
  - `/contact/` にmeta descriptionが未設定
  - `/content/` にmeta descriptionが未設定
  - `/phimosis/surgery/` のmeta descriptionもトップページとほぼ同一
- **影響:** クリック率の低下、検索結果での差別化ができない
- **改善アクション:** 各ページ固有のdescriptionを120文字程度で設定

#### A-2. openingHoursSpecificationの値が空
- **確認済み事実:** 全院ページのMedicalClinicスキーマでopeningHoursSpecificationのopens/closes値が空白
- **影響:** Googleマップ・ナレッジパネルで営業時間が正しく表示されない
- **改善アクション:**
```json
"openingHoursSpecification": [
  {
    "@type": "OpeningHoursSpecification",
    "dayOfWeek": ["Monday","Tuesday","Wednesday","Thursday","Friday","Saturday","Sunday"],
    "opens": "10:00",
    "closes": "20:00"
  }
]
```

#### A-3. Productスキーマのprices有効期限切れ
- **確認済み事実:** priceValidUntilが `2025-12-31` で設定されており期限切れ
- **影響:** Googleがリッチリザルトを非表示にする可能性、構造化データのバリデーションエラー
- **改善アクション:** priceValidUntilを将来の日付（例: `2027-12-31`）に更新するか、価格が固定であれば削除を検討

#### A-4. サイトマップのlastmodが古い
- **確認済み事実:** サイトマップ内のlastmodが2020-2021年で停止。実際のページ更新日（schema上2026年）と大幅に乖離
- **影響:** Googleのクロール効率が低下、新しいコンテンツの発見が遅れる
- **改善アクション:** WordPress sitemapプラグイン（Yoast SEO、XML Sitemaps等）で自動生成に切り替え

#### A-5. FAQのURL構造が二重化
- **確認済み事実:** 同カテゴリのFAQが2つのパスに存在
  - `/faq/phimosis/` と `/phimosis/faq/`
  - `/faq/powerup/` と `/powerup/faq/`
  - `/faq/trouble/` と `/trouble/faq/`
  - `/faq/price/` と `/price/faq/`
- **影響:** コンテンツカニバリゼーション、リンクジュースの分散
- **改善アクション:** 片方のURLにcanonicalを集約するか、301リダイレクトで統一

### 深刻度B（改善推奨）

#### B-1. MedicalClinicスキーマのtelephoneが共通フリーダイヤル
- **確認済み事実:** 全院のMedicalClinicスキーマでtelephone属性が `+81120127411`（共通フリーダイヤル）に統一
- **影響:** GMBとの整合性問題、各院の個別認識が弱くなる
- **改善アクション:** 各院の個別電話番号をtelephone属性に設定し、フリーダイヤルは別途contactPoint等で記載

#### B-2. 画像のalt属性が不完全
- **確認済み事実:** ロゴ等の主要画像にはalt属性あり、装飾画像やCTAボタン画像にalt属性なし
- **影響:** アクセシビリティの低下、画像検索での機会損失
- **改善アクション:** 全画像にコンテキストに合ったalt属性を追加（装飾画像は`alt=""`を明示）

#### B-3. 画像のlazy loadingが未実装
- **確認済み事実:** `loading="lazy"` 属性が画像要素に確認されなかった
- **影響:** ファーストビュー外の画像が初回ロード時に全て読み込まれ、表示速度が低下
- **改善アクション:**
```html
<img src="image.jpg" loading="lazy" alt="説明文">
```

#### B-4. robots.txtの書式
- **確認済み事実:** `User-Agent:*` とスペースなしで記述
- **影響:** 多くのクローラーは許容するが、仕様上は `User-Agent: *` が正式
- **改善アクション:**
```
User-Agent: *
Allow: /

Sitemap: https://www.norst.co.jp/sitemap.xml
```

#### C-1. canonicalタグの一部未確認
